# Integration Tests Rewrite — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the shell-script integration tests with Go tests using testcontainers-go and the miekg/dns client, using unpoller types for the mock controller.

**Architecture:** Docker compose orchestrates two containers: a mock UniFi controller (using unpoller types and API path constants for realistic responses) and CoreDNS compiled with our plugin. Go integration tests (gated by `//go:build integration`) use testcontainers-go compose module to start the stack and miekg/dns to query CoreDNS and assert results. Dockerfiles are optimized for layer caching.

**Tech Stack:** testcontainers-go (compose module), miekg/dns, unpoller/unifi types, Docker

### Task 4: Optimize CoreDNS Dockerfile

**Files:**
- Rewrite: `integration/Dockerfile.coredns`

- [ ] **Step 1: Write optimized Dockerfile**

```dockerfile
FROM golang:1.22-alpine AS builder
RUN apk add --no-cache git

# Clone CoreDNS and inject plugin — these layers are stable
RUN git clone --depth 1 --branch v1.11.3 https://github.com/coredns/coredns.git /coredns
WORKDIR /coredns
RUN sed -i '/^forward:forward/a unifi:github.com/jhogendorn/coredns-unifi' plugin.cfg

# Copy plugin module files first for dependency caching
COPY go.mod go.sum /plugin/
RUN go mod edit -replace github.com/jhogendorn/coredns-unifi=/plugin && \
    go generate && \
    go get github.com/jhogendorn/coredns-unifi && \
    go mod download

# Now copy plugin source and build
COPY *.go /plugin/
RUN go build -o coredns .

FROM alpine:3.20
COPY --from=builder /coredns/coredns /usr/local/bin/coredns
EXPOSE 53 53/udp
CMD ["coredns", "-conf", "/etc/coredns/Corefile"]
```

Key optimizations:
- CoreDNS clone + plugin.cfg injection cached (rarely changes)
- Plugin `go.mod`/`go.sum` copied first, then `go generate` + `go get` + `go mod download` — dependency layer cached
- Plugin `*.go` source copied last — only the final build step re-runs on source changes
- Removed `make` from apk (not used)

Expected: Both containers build successfully

### Task 6: Write Go integration tests

**Files:**
- Create: `integration_test.go`

**Context:** This file uses build tag `//go:build integration` so it only runs when explicitly requested. It uses testcontainers-go compose module to start the Docker stack and miekg/dns to make queries.

- [ ] **Step 1: Write integration_test.go**

The file must:
1. Use `//go:build integration` build tag
2. Be in `package unifi` (same as the rest of the project)
3. Import `github.com/testcontainers/testcontainers-go/modules/compose`, `github.com/miekg/dns`, `github.com/testcontainers/testcontainers-go/wait`
4. Have a `TestMain` or helper function that:
   - Creates compose stack from `integration/docker-compose.yml`
   - Waits for mock-controller (HTTP health on 8443) and coredns (listening on 53/udp)
   - Gets the mapped port for coredns on 53/udp
   - Defers `stack.Down()`
5. Have a DNS query helper: `queryA(t *testing.T, server string, name string) (string, int)` that returns the resolved IP and rcode using `miekg/dns.Client`
6. Have these test functions:
   - `TestNamedClientResolves` — desktop.home.lan → 192.168.1.10, laptop.home.lan → 192.168.1.11
   - `TestHostnameFallback` — phone-dhcp.home.lan → 192.168.1.12 (Name empty, falls back to Hostname)
   - `TestMultiNetworkDomain` — iot-sensor.iot.lan → 10.0.0.50
   - `TestSkipsNamelessClient` — query for ".home.lan" returns no answer (client5 has no name or hostname)
   - `TestNXDOMAIN` — nonexistent.home.lan → NXDOMAIN rcode
   - `TestCaseInsensitive` — DESKTOP.home.lan → 192.168.1.10

**Implementation pattern:**
```go
func TestNamedClientResolves(t *testing.T) {
    ip, rcode := queryA(t, dnsAddr, "desktop.home.lan.")
    if rcode != dns.RcodeSuccess {
        t.Fatalf("expected NOERROR, got %s", dns.RcodeToString[rcode])
    }
    if ip != "192.168.1.10" {
        t.Errorf("expected 192.168.1.10, got %s", ip)
    }
}
```

**Compose setup pattern:**
Use `compose.NewDockerCompose("integration/docker-compose.yml")` with `WaitForService` strategies:
- mock-controller: `wait.ForHTTP("/status").WithPort("8443/tcp")`
- coredns: `wait.ForListeningPort("53/udp").WithStartupTimeout(30*time.Second)`

Then get the coredns mapped port via `stack.ServiceContainer(ctx, "coredns")` → `container.MappedPort(ctx, "53/udp")`.

Store `dnsAddr` as a package-level var set in `TestMain`.
